# 🤖 Asistent Inteligent RAG: Oracle AI Vector Search & Modele Locale

Acest proiect implementează un sistem avansat de **Retrieval-Augmented Generation (RAG)** capabil să proceseze documente PDF, să le vectorizeze folosind capabilitățile AI ale bazei de date Oracle și să genereze răspunsuri inteligente folosind modele de limbaj (LLM) rulate local.

Aplicația oferă o interfață web intuitivă dezvoltată în **Streamlit**, structurată pe trei module principale:
1. **💬 Chat cu Documentele:** Conversație contextuală cu un asistent virtual.
2. **🔍 Căutare Semantică:** Motor de regăsire a fragmentelor relevante pe baza sensului, nu doar a cuvintelor cheie.
3. **📁 Gestionare Documente:** Modul pentru încărcarea, curățarea și indexarea fișierelor PDF noi.

---

## 🏗️ Arhitectura Soluției

Sistemul urmează un pipeline modular, optimizat pentru performanță locală și securitatea datelor:

1.  **Ingestia Datelor:** Extragerea textului din PDF-uri folosind `pdfplumber`.
2.  **Preprocesare & Curățare:** Eliminarea automată a numerelor de pagină, repararea cuvintelor despărțite în silabe (`- \n`) și normalizarea spațiilor libere prin Regex.
3.  **Segmentare (Chunking):** Împărțirea textului în fragmente de ~900 de caractere, cu un **overlap glisant de 150 de caractere** pentru a păstra coerența semantică între segmente.
4.  **Vectorizare (Embedding):** Transformarea textului în vectori de 384 de dimensiuni folosind modelul multilingv `paraphrase-multilingual-MiniLM-L12-v2`.
5.  **Stocare (Vector DB):** Salvarea vectorilor și a metadatelor (nume document, pagină) în **Oracle Database 23ai**.
6.  **Regăsire (Retrieval):** Căutare hibridă (Vectorială prin distanță Cosinus + Fallback Lexical).
7.  **Generare (LLM):** Formularea răspunsului final folosind modelul causal **Qwen 2.5 (1.5B Instruct)**, rulat local prin `transformers`.

---

## 📊 Modelul de Date

Sistemul utilizează un model de date optimizat pentru căutarea vectorială în Oracle. Structura tabelului principal este următoarea:

```
sql
CREATE TABLE document_chunks2 (
    id          NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY PRIMARY KEY,
    doc_name    VARCHAR2(255),    -- Numele prietenos al articolului
    page_number NUMBER,           -- Pagina de unde a fost extras fragmentul
    line_number NUMBER,           -- Linia de start (pentru referință)
    text_chunk  CLOB,             -- Conținutul textual brut
    embedding   VECTOR(384, FLOAT32) -- Reprezentarea matematică a sensului
);
```

Indexare: Pentru a asigura un timp de răspuns sub-secundă, este aplicat un index de tip NEIGHBOR GRAPH peste coloana de vectori.

## 📈 Execuție și Rezultate
**Interpretarea Rezultatelor**
Sistemul a evoluat de la o arhitectură bazată pe FLAN-T5 (care prezenta dificultăți în urmărirea instrucțiunilor și în limba română) la una bazată pe Qwen 2.5.

- Acuratețe: Prin folosirea modelului Causal, răspunsurile sunt acum formulate în propoziții cursive, respectând contextul oferit.

- Citații: Fiecare răspuns include sursa (ex: [Healthy_diet.pdf p.2]), eliminând fenomenul de halucinație.

- Performanță Căutare: Căutarea semantică reușește să găsească concepte înrudite (ex: cauți "salinitate" și găsește paragrafe despre "salt intake") datorită modelului MiniLM.
